ARM Registers and Instruction Format

Briefly describe each ARM CPU register, including the status register (17 total).

For the status register, please also describe the Z, C, N, and V bits.

R0 - R15: General Purpose registers for holding values to compute on.

R13: Called the Stack Pointer (SP), used to tore the location of the top of the stack.

R14: Called the Link Register (LR), used to store the program counter a function should return to after it completes execution.

R15: Called the Program Counter (PC), used to store the memory address of the current instruction being executed.

Status Register: This register gives information about the results of the last instruction executed. It is made up of a number of status flags. Four of which are:

  • Zero Flag (Z): Z is 1 if the result of an instruction is 0.
  • Carry Flag (C): C is 1 if the result of an instruction causes a carry or unsigned overflow to occur.
  • Negative Flag (N): N is 1 if the result of an instruction is negative (i.e. bit 31 is 1).
  • Overflow Flag (V): V is 1 if the result of an instruction caused a signed overflow.
