General ARM Assembly

Give a brief description of each of the seven addressing modes.

Immediate Offset Addressing Mode: The location loaded from or stored to is the value of a "base-address" register, plus the value of a constant. Additionally, the "base-address" register can optionally be incremented or decremented after the load / store completes.

Regsister Offset Addressing Mode: The location loaded from or stored to is the value of a "base-address" register, plus the value of an "offset" register.

PC-Relative Addressing Mode: The location loaded from or stored to is the value in the PC register +/- a constant value.

Load / Store Multiple Registers Addressing Mode: Loads or stores a list of registers from or to a sequential location in Data Memory.

PUSH and POP Register Addressing Mode: PUSH and POP are used to place or remove data from the top of the stack, which is located in Data Memory.

Load / Store Exclusive Addressing Mode: A mode that is used to help guarantee one process at a time is accessing a register (typically used for multi-process systems)

Inherent Addressing Mode: Not really an addressing mode, just means that register names are specified as inputs to be operated on (i.e. Data Memory is not accessed).
